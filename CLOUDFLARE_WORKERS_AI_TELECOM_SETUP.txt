광석이네 통신방 - Cloudflare Workers AI 설정 방법

이 버전은 Firebase Cloud Functions/OpenAI를 쓰지 않습니다.
GitHub Pages 화면(app.js)이 Cloudflare Worker를 호출하고,
Cloudflare Worker가 Workers AI로 대사를 생성한 뒤,
브라우저가 그 대사를 Firestore에 저장합니다.

1) Cloudflare 로그인
- https://dash.cloudflare.com 접속
- Workers & Pages 메뉴로 이동

2) Worker 만들기
- Create application 또는 Create Worker 클릭
- 이름 예시: kks-telecom-ai
- 기본 코드가 나오면 cloudflare-worker-ai-telecom.js 내용을 전부 붙여넣기

3) Workers AI binding 추가
- Worker 설정(Settings) 또는 Bindings/Variables로 이동
- AI binding 추가
- Binding name은 반드시 AI 로 설정
- Workers AI 사용이 계정에서 활성화되어 있어야 함

4) 배포
- Deploy 클릭
- 배포 후 주소를 복사
  예: https://kks-telecom-ai.내서브도메인.workers.dev

5) app.js에 Worker 주소 넣기
- app.js에서 아래 줄을 찾기
  const FB_TELECOM_AI_WORKER_URL = "https://kks-telecom-ai.kos20050627.workers.dev";
- 복사한 Worker 주소로 바꾸기
  const FB_TELECOM_AI_WORKER_URL = "https://kks-telecom-ai.내서브도메인.workers.dev";

6) GitHub에 정적 파일 업로드
- index.html
- app.js
- style.css
- cloudflare-worker-ai-telecom.js는 GitHub에 올려도 되지만, 실제 실행은 Cloudflare Worker에 붙여넣은 코드가 담당함

7) Firestore 규칙 배포/적용
- firestore.rules에 통신방 member generated 저장 허용 규칙이 들어 있음
- 기존 규칙이 있다면 덮어쓰지 말고, chatRooms 부분만 합치는 것이 안전함

주의
- Cloudflare Workers AI에도 무료/유료 한도와 사용량 제한이 있음.
- Worker URL은 공개될 수 있으므로, 사용량이 늘면 Cloudflare 대시보드에서 제한/모니터링을 확인해야 함.
- 이 구조는 간단화를 위해 Worker가 Firestore에 직접 쓰지 않고, 브라우저가 로그인 사용자 권한으로 저장함.

[2026-05-30 추가 수정]
- 무료 자동 멤버 반응은 사용하지 않습니다.
- Cloudflare Workers AI가 최근 대화와 반복 금지 문장을 함께 받아 새 대사를 생성합니다.
- 방이 조용하면 70~140초 간격으로 AI가 먼저 짧은 잡담을 생성합니다.
- 비용/무료 할당량 보호를 위해 한 접속 세션당 자동 선발화은 최대 12회로 제한했습니다.
- Worker 코드를 다시 Cloudflare의 Edit code/Quick edit에 붙여넣고 Save and deploy 해야 반영됩니다.
